Submit to App Store

eas submit takes a build you already produced (Section 12) and uploads it directly to App Store Connect, using credentials EAS manages for you — no dragging files into Xcode's Transporter app.

Run the submission pipeline yourself Press start and watch each real stage of an App Store submission light up in order.

Apple processing
Waiting for review
In review
Ready for sale

terminal 
eas submit --platform ios --latest
eas.json 
{
  "submit": {
    "production": {
      "ios": {
        "appleId": "you@example.com",
        "ascAppId": "1234567890",
        "appleTeamId": "AB12CD34EF"
      }
    }
  }
}
  • --latest — automatically submits the most recent successful production build, so you don't need to hunt for a build ID.
  • ascAppId — your app's unique App Store Connect identifier, found in App Store Connect once your app record is created there.
  • appleTeamId — identifies which Apple Developer team account this submission belongs to, important if you manage multiple client apps.
Apple's review process is still manual on their end eas submit automates the upload, but Apple's human/automated review of your actual app content still applies — typically anywhere from hours to a couple of days. Nothing about EAS Submit bypasses or speeds up Apple's own review.
Beginner mistakeNot creating the app record in App Store Connect first — ascAppId requires an app entry to already exist there before you can submit to it.
Intermediate mistakeSubmitting a build with an unchanged build number after a rejection fix — Apple requires a genuinely new, incremented build for every resubmission.

Quick check

Does eas submit speed up Apple's app review itself?

Correct — EAS Submit removes the manual upload friction, but the actual review by Apple happens exactly as it would for any submission.

Submit to Google Play

Google Play's submission model is different in one important way: instead of a single review queue, releases go through tracks that control exactly who receives them, with fine-grained rollout percentages even within the production track.

Try it: pick a track and see what it means

Internal testing: Up to 100 hand-picked testers, updates appear almost instantly, no Google review required — the fastest feedback loop.

Staged rollout: try it

Even within the production track, you rarely release to 100% of users immediately. Drag the slider to see how a staged rollout limits exposure if something's wrong.

Rollout percentage20%
~20 out of every 100 users updating right now get the new version; the rest stay on the previous one until you increase the percentage.

Why stage a rollout at all

If a crash-inducing bug ships, a 5% rollout means 95% of your users never see it. You can halt the rollout entirely from the Play Console before increasing further.

terminal 
eas submit --platform android --latest
eas.json 
{
  "submit": {
    "production": {
      "android": {
        "serviceAccountKeyPath": "./google-service-account.json",
        "track": "internal"
      }
    }
  }
}
  • serviceAccountKeyPath — Android submission auth uses a downloadable service account JSON key from Google Cloud, rather than a personal login like Apple's flow.
  • "track": "internal" — controls which of the four tracks this submission targets; change to "production" once ready for a real release.
TrackAudienceGoogle review?
Internal testingUp to 100 chosen testersNo
Closed testingSpecific groups/email listsLight review
Open testingAnyone who opts in via a public linkFull review
ProductionAll Play Store users, with optional staged rolloutFull review
Beginner mistakeReleasing straight to 100% production for a major update instead of using a staged rollout to catch issues early.
Intermediate mistakeLosing track of the Google Cloud service account key, then having to regenerate submission credentials mid-release cycle.

Section 13 final check

What's the main benefit of a staged rollout on the production track?

Correct — a staged rollout contains the blast radius of a bad release, which is exactly why it's a standard production practice.
